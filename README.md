# go_practice

#Go 言語の特徴
**メリット**

- C や JAVA と構文が似ている
- 直接機械語に変換されるから処理速度が速い
- 処理が簡易的で複数処理を同時に行うことができる
- ライブラリが豊富
- 安全性が高い(メモリの開放忘れなどが発生しにくい)

**デメリット**

- Genrics が存在しない(引数の関数の型がきまっている)
- 例外処理がない
- コードの継承がない(オブジェクト指向言語ではなく関数型プログラム言語)

#ビルド実行コマンド

- go build ファイル名.go → .\ファイル名(go build でできたファイル)
- go run ファイル名.go(デバック時に使用)
